1. ບົດນຳ (Introduction)

ໃນປັດຈຸບັນ, ລະບົບການສຶກສາພວມກຳລັງຫັນປ່ຽນໄປສູ່ຍຸກດິຈິຕອນ ທີ່ມີການນຳໃຊ້ເຕັກໂນໂລຊີ ແລະ ລະບົບຖານຂໍ້ມູນເຂົ້າໃນການຄຸ້ມຄອງການຮຽນ-ການສອນຢ່າງກວ້າງຂວາງ. ການປ່ຽນແປງນີ້ເຮັດໃຫ້ມີຂໍ້ມູນດ້ານການສຶກສາ (Educational Data) ຖືກສະສົມໄວ້ເປັນຈຳນວນຫຼວງຫຼາຍ ເຊິ່ງສ່ວນໃຫຍ່ຍັງບໍ່ທັນໄດ້ຖືກນຳມາສະກັດ ຫຼື ວິເຄາະໃຫ້ເກີດປະໂຫຍດສູງສຸດ. ບັນຫາທີ່ພົບເລື້ອຍໃນສະຖາບັນການສຶກສາແມ່ນ ອັດຕາການຕົກຄ້າງ, ການປະລະການຮຽນຂອງນັກສຶກສາ (Student Dropout), ແລະ ຜົນການຮຽນທີ່ບໍ່ເຖິງເກນມາດຕະຖານ ເຊິ່ງບັນຫາເຫຼົ່ານີ້ບໍ່ພຽງແຕ່ສົ່ງຜົນກະທົບຕໍ່ອະນາຄົດຂອງໂຕນັກສຶກສາເອງເທົ່ານັ້ນ ແຕ່ຍັງສົ່ງຜົນຕໍ່ຄຸນນະພາບ ແລະ ຊື່ສຽງຂອງສະຖາບັນການສຶກສາອີກດ້ວຍ. ທີ່ຜ່ານມາ, ການຕິດຕາມ ແລະ ຊ່ວຍເຫຼືອນັກສຶກສາທີ່ມີຄວາມສ່ຽງຕໍ່ບັນຫາດັ່ງກ່າວ ມັກຈະເຮັດແບບຕັ້ງທ່າຮັບ ຄື ລໍຖ້າໃຫ້ຜົນການຮຽນອອກມາບໍ່ດີກ່ອນ ຈຶ່ງມີການແນະນຳ ເຊິ່ງມັນຈະຊ້າເກີນໄປ ແລະ ບໍ່ທັນການ.

ດ້ວຍເຫດນີ້, ການນຳໃຊ້ເຕັກນິກ ການຂຸດຄົ້ນຂໍ້ມູນ (Data Mining) ແລະ ການຮຽນຮູ້ຂອງເຄື່ອງຈັກ (Machine Learning) ເຂົ້າໃນການພະຍາກອນຜົນການຮຽນຂອງນັກສຶກສາ (Student Academic Performance Prediction) ຈຶ່ງມີຄວາມສຳຄັນ ແລະ ເປັນແນວທາງຮູບແບບໃໝ່ທີ່ມີປະສິດທິພາບສູງ. ການວິເຄາະຂໍ້ມູນຮູບແບບນີ້ ຈະຊ່ວຍຄົ້ນຫາປັດໄຈຕ່າງໆ ທີ່ມີອິດທິພົນຕໍ່ຜົນການຮຽນ ບໍ່ວ່າຈະເປັນ ປະຫວັດການຮຽນ, ພຶດຕິກຳການມາຮຽນ, ຫຼື ປັດໄຈທາງດ້ານສັງຄົມ ແລະ ອື່ນໆ.

ຄວາມສຳຄັນຂອງວຽກງານນີ້ ຈະຊ່ວຍປ່ຽນຮູບແບບການບໍລິຫານການສຶກສາຈາກ "ການຕັ້ງທ່າຮັບ" ມາເປັນ "ການປ້ອງກັນ ແລະ ຄາດການລ່ວງໜ້າ". ຜົນປະໂຫຍດທີ່ໄດ້ຮັບແມ່ນສາມາດແບ່ງອອກເປັນ 3 ສ່ວນຫຼັກ ຄື:

  1. ດ້ານອາຈານ ແລະ ຜູ້ບໍລິຫານ: ສາມາດຄາດຄະເນກຸ່ມນັກສຶກສາທີ່ມີຄວາມສ່ຽງ (At-risk students) ໄດ້ຕັ້ງແຕ່ຫົວທີ ເຮັດໃຫ້ສາມາດວາງແຜນຊ່ວຍເຫຼືອ, ປັບປຸງຫຼັກສູດ, ຫຼື ໃຫ້ຄຳປຶກສາໄດ້ຢ່າງຖືກຈຸດ ແລະ ທັນການ.
  2. ດ້ານນັກສຶກສາ: ໄດ້ຮັບການແຈ້ງເຕືອນ ແລະ ປັບປຸງພຶດຕິກຳການຮຽນຂອງຕົນເອງເພື່ອຫຼີກລ້ຽງການຕົກວິຊາ ຫຼື ການປະລະການຮຽນ.
  3. ດ້ານສະຖາບັນ: ຊ່ວຍຍົກລະດັບມາດຕະຖານການສຶກສາ, ເພີ່ມອັດຕາການຈົບການສຶກສາ, ແລະ ບໍລິຫານຊັບພະຍາກອນມະນຸດໄດ້ຢ່າງມີປະສິດທິພາບ.

ດັ່ງນັ້ນ, ງານວິໄຈສະບັບນີ້ຈຶ່ງມຸ່ງສຶກສາ ແລະ ພັດທະນາແບບຈຳລອງການພະຍາກອນຜົນການຮຽນ ເພື່ອເປັນເຄື່ອງມືສະໜັບສະໜູນການຕັດສິນໃຈ ໃຫ້ແກ່ສະຖາບັນການສຶກສາໃນການພັດທະນາຄຸນນະພາບການຮຽນ-ການສອນໃຫ້ມີຄວາມຍືນຍົງໃນຍຸກດີຈີຕອນ.

3. ຊຸດຂໍ້ມູນທີ່ໃຊ້ (Data set)

ຊຸດຂໍ້ມູນທີ່ນຳໃຊ້ໃນການວິໄຈຄັ້ງນີ້ ແມ່ນໄດ້ມາຈາກລະບົບຖານຂໍ້ມູນການສຶກສາໃນອິນເຕີເນັດ (kaggle.com & worldbank.org) ທີ່ມີ 5200 ລາຍການ (record), 18 features ເຊິ່ງປະກອບດ້ວຍຂໍ້ມູນຂອງບັນດາ feature ທີ່ກ່ຽວຂ້ອງກັບນັກສຶກສາ ແລະ ເປັນຊຸດຂໍ້ມູນທີ່ສົມບູນແບບ (ບໍ່ມີຂໍ້ມູນຊ້ຳຊ້ອນ, ບໍ່ມີ missing value, ...),

ໂຄງສ້າງຂອງຊຸດຂໍ້ມູນ ໄດ້ຖືກຄັດເລືອກ ແລະ ແບ່ງອອກເປັນ 3 ກຸ່ມປັດໄຈຫຼັກ ທີ່ມີຜົນຕໍ່ການຮຽນ ຄື: (1) ຂໍ້ມູນສ່ວນຕົວຂອງນັກສຶກສາ ເຊັ່ນ: ເພດ, ອາຍຸ, ລະດັບການສຶກສາຂອງຜູ້ປົກຄອງ, ..., (2) ປະຫວັດການສຶກສາ ເຊັ່ນ: ຄະແນນກາງພາກ, ຄະແນນຂອງປີຜ່ານມາ, ແລະ (3) ພຶດຕິກຳການຮຽນ ເຊັ່ນ: ອັດຕາການມາຮຽນ, ການສົ່ງວຽກບ້ານ, ເວລາຮຽນນອກໂມງ, ...

ຊຸດຂໍ້ມູນປະກອບດ້ວຍບັນດາປະເພດຂໍ້ມູນຂອງ Attribute/Features ຄື: Nominal (ເພດ: ຍິງ, ຊາຍ), Ordinal (ເກຣດ: A, B, C, D), ແລະ Internal (ຄະແນນ: 0-100), ເປັນຂໍ້ມູນປະເພດຕົວເລກ ແລະ ຂໍ້ຄວາມ.

2. Data Cleaning

ກິດຈະກຳ: ກວດສອບ ແລະ ຈັດການກັບຂໍ້ມູນທີ່ບໍ່ສົມບູນ (ຊຸດຂໍ້ມູນນີ້ ສົມບູນເກືອບ 100%).

ວິທີການ: ກວດສອບຄ່າທີ່ຂາດຫາຍ (Missing Values), ກວດຂໍ້ມູນທີ່ຊ້ຳຊ້ອນ ຫຼື ຜິດປົກກະຕິ (Outliers), ແລະ ເຮັດການທຳຄວາມສະອາດຂໍ້ມູນ

3. Data Preprocessing

ກິດຈະກຳ: ປັບແຕ່ງ ແລະ ແປງຮູບແບບຂໍ້ມູນໃຫ້ພ້ອມສຳລັບການປະມວນຜົນ.

ວິທີການ: ປ່ຽນຂໍ້ມູນໂຕໜັງສືໃຫ້ເປັນໂຕເລກ (ເຊັ່ນ: ເພດ) ແລະ ປັບມາດຕະຖານ (Scaling) ໃຫ້ຄະແນນທຸກວິຊາຢູ່ໃນເກນດຽວກັນ.

4. Feature Selection

ກິດຈະກຳ: ຄັດເລືອກເອົາສະເພາະຕົວແປ (feature) ທີ່ເປັນປັດໄຈຫຼັກຕໍ່ຜົນການຮຽນ.

ວິທີການ: ຕັດຕົວແປ (feature) ທີ່ບໍ່ກ່ຽວຂ້ອງອອກ (ເຊັ່ນ: ລະຫັດນັກສຶກສາ) ເພື່ອໃຫ້ Model ປະມວນຜົນໄດ້ໄວ ແລະ ຊັດເຈນຂື້ນ.

Supervised Learning.

RandomForestClassifier:
  1. ເປັນ algorithm ທີ່ນິຍົມໃຊ້ ໃນສາຍ ການຈັດຈຳແນກຂໍ້ມູນ (data classification)
  2. ມີຄວາມຊັດເຈນ (ເມັ້ນຢຳ) ສູງ
  3. ຫຼຸດຜ່ອນການມີ overfitting
  4. ເໝາະສົມກັບຊຸດຂໍ້ມູນການສຶກສາ
ExtraTreesClassifier:
  1. Faster training, Good Performance
  2. ມີຄວາມສະຖຽນຕໍ່ວິທີການເຮັດຈັດຈຳແນກຂໍ້ມູນ (Stable for classification tasks)
  3. ຮອງຮັບຊຸດຂໍ້ມູນຈຳນວນຫຼາຍ (large datasets)
  4. ມີຄວາມສະຖຽນຕໍ່ຂໍ້ມູນຜິດປົກກະຕິ (Robust to outliers)
DecisionTreeClassifier:
  1. ເປັນ algorithm ທີ່ງ່າຍ (ເຂົ້າໃຈງ່າຍ)
  2. ມີຄວາມໄວສູງຕໍ່ຊຸດຂໍ້ມູນທີ່ບໍ່ຫຼາຍ
